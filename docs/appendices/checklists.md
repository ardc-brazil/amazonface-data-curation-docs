# Checklists

These checklists are execution aids for the protocols described in the main chapters. They are intentionally concise, but they should be used together with the narrative guidance in the foundations, pipeline, and operations sections. A checked item means that evidence exists, not merely that the task was discussed informally.

## Dataset Mapping Checklist

- Dataset working title defined.
- Scientific area assigned.
- Data mentor identified.
- Operational steward identified.
- Lifecycle state assigned.
- Data type and format class recorded.
- Current storage location recorded.
- Access status recorded.
- Target product level recorded.
- Known gaps, anomalies, or restrictions recorded.

## Responsible Contact Checklist

- Initial contact sent.
- Ownership confirmed.
- Diagnostic meeting completed when needed.
- Acquisition unit confirmed.
- Temporal and spatial coverage confirmed.
- Access/embargo decision confirmed.
- Required metadata fields collected.
- Publication/DOI expectations recorded.

## Training Checklist

- Steward trained on WebDAV/RaiDrive or approved transfer tool.
- Landing-zone path confirmed.
- Raw-upload rule explained.
- Metadata template explained.
- Large-file handling explained when relevant.
- Upload completion notification procedure explained.
- Incident reporting channel confirmed.

## Onboarding Checklist

- Dataset identified and assigned mentor.
- Minimum metadata submitted.
- Lifecycle state classified (legacy/active/campaign).
- Datastream boundary validated.

## Ingestion Checklist

- Files transferred to controlled landing zone.
- Integrity checks executed.
- Naming standardization validated.
- Ingestion evidence logged.
- File count and logical size recorded.
- Temporal coverage validated.
- Access restrictions checked.
- Quarantine decision recorded if needed.

## QC and Curation Checklist

- Product level target defined (00/a1/b1/c1).
- QC checks executed (range, delta/spike, missing/invalid).
- Metadata enriched and validated.
- Provenance links preserved.
- Level 00 README completed.
- DataCite JSON created when publication is expected.
- Processing script or workflow included.
- Inclusion/exclusion rules documented.
- Mentor review completed.

## WebDAV/Data Lake Checklist

- Landing, archive, standard, derived, and publish paths defined.
- Level 00 archived under authoritative raw storage.
- Processed products stored outside raw archive.
- Hard-link usage documented if applicable.
- Backup/integrity evidence recorded.
- Actual storage path documented in README or mapping sheet.

## Release Checklist

- Gate approvals completed.
- Version and changelog prepared.
- License and citation defined.
- Publication record generated.
- DataMap metadata validated.
- DOI status recorded.
- DataMap URL recorded.
- Release date recorded.
- Post-publication maintenance owner assigned.
